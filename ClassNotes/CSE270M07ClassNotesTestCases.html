CSE 270: Welcome to Week 7: Test cases
This week you are going to learn how to write test cases. Writing test cases is a simple task in some cases and a complete task in other cases.
For Teach 07, we will start with simple cases. I have provided a list of simple tasks that you need to figure out how to verify. Â You will have to define the requirement that you are verifying, which is a bit more than just the story I provided, or you can provide your own story. Provide the steps, and how you know you have completed the story, what the evidence is. Yes, if you think this is like the scientific method, you are correct. The requirement is the thesis, the test steps are the experiment, and the verification is the conclusion.
I have found that there is a simple format for writing test steps as a programmer or tester. A tester or user does something, and then the software reacts. So you write a “step” action and a “results” statement. Instead of the “results,” you write it as a “verification” statement when you get to the end. The “step/results” lead up or set up the system to be “step/verified.”
For the Lab, you are working with Dr. Mobile again, designing four tests. You will find the link to Dr. Mobile's application HTML file in the assignment, and you will notice that the Sensors are not installed yet, and this assignment is a follow-on to the week 05 lab assignment. Because the project is developing the software independently of the hardware, you are writing the tests without the hardware available. You can use simulated equipment like your phone, computer, tablet, or any other electronic device. Make sure you use the “step/results|verified” method. Make sure you reply to your classmate's post as part of the review process.

For the Ponder assignment, you will write other tests using the “step/results|verified” method. You will base them on different methods of testing:

  • Manual (which is what you did for the Lab),
  • Automated (you can use selenium if you want, but providing a paper version of a plan is find)
  • Analysis
  • Demo
  • Inspection

Feel free to use your imagination, and provide your assumptions.
