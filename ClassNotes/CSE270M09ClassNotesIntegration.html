CSE 270: Classroom Notes M09: Regression Testing

Notes for Classroom

Overview

As software is being developed, many times it is divided into different pieces, top-down design, and worked independently. As some point the piece need to come back together. When all the pieces come together, the problems really begin to pile up. Much like making bread, you can’t put all the ingredients in the mixer at the same time and expect perfect bread. Assembly of software is a key component to delivering on time. Understanding how to describe the inputs getting processed to output throughout the entire workflow is a vital skill in delivery the desired product. In addition to making sure that product is comes together, there is also making sure it works with other 3rd party components, interfaces, and standards.

  • Students will understand the risks of Integration Testing
  • Students explore the different parts of a system (End to End)
  • Students will organize and create an Integration Plan and then test it using QA Checklist

Objectives

By the end of the week, each student will be able to:

  • Implement Testing Applications (Cyber, UX, Full-Stack, End-To-End)
  • Build an Integration Plan

Announcement

Welcome to Week 09: Integration Testing
This week you will be learning about Integration Tests.
There are essentially four different types of tests.

  • Unit Tests check the implementation for errors at each unit of work. As each unit is finished it is added to the whole.
  • Integration Tests grow as the units are finished. If not organized correctly, this can be a nightmare quickly. Usually, a plan is needed to keep them straight.
  • System Tests are end-to-end tests and verify that the entire system has been fully integrated. In larger projects, an extension of system tests is Acceptance Tests are tests. Acceptance tests show to the customer that the system validates their needs.
  • Regression Test is a series of happy path tests. Each time Units are added or changed the system needs to be tested to verify most of the system still works, and the change or new capability has not introduced new bugs.

Image, for a moment you own a car if you do not already have one. The Unit Tests are the individual parts of the car. For example, tires go through a thorough examination before they are installed. An Integration Test is the test drive and performance tests on the track after the car is assembled. The System or Acceptance test the customer test driving the car at the car dealer. The Regression test is the 14-to-30-point inspection that the car mechanic does when you bring it in for service.
This week you will be developing an Integration Plan.  This is like a report and not a set of tests. Since we are working with a small application, the plan is essentially an outline to keep track of the progress of integrating the units and testing them.
Please follow the updated outline for the plan. You will be using parts from the previous assignment to assemble the plan. So, this is equivalent to a midterm.
