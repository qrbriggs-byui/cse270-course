CSE 270 Classroom Notes M06: Testing Tools

Notes for Classroom

Overview

Testing is a critical phase just before delivering the software out the door (DevOps). Too many times the phase is squeezed. Finding solutions to work faster in these times helps with the stress. Different tools are available to help practitioners work more efficiently and faster in S/W Testing. Having the right tool for the right job is a sign of a professional. Many times, knowing how to make tools when none are available is a valuable skill.

  • Students will get an overview of DevOps, and a testers role in the delivery of software.
  • Students will identify and judge testing tools, software and workflows

Objectives

By the end of the week, each student will be able to:

  • Understand the critical role testing has in DevOps
  • Evaluate Testing Tools

Announcement

Welcome to Week 06 Testing Tools.
There are a ton of testing tools, from DevOps tools (Azure, Jenkins) to Testing Platforms(Protector, Selenium, Telerik, Mocha) to Testing Programs (Angular, Junit). Each of these tools assists tester in automating and increasing performance. They don’t necessarily help find the bugs. Finding bugs is the difficult part of testing, and why we had discussed the methods of testing first before we discussed the cool tools.
This week we are going to mention the cool tools. Your team will be assigned a tool to research and create a post about the tool.
For the lab, you will also install Microsoft Visual Studio and install Selenium on your computer. We will be using this for later labs.

As part of this assignment, you need to determine if Selenium is the best choice for your project.
