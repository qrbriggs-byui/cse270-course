CSE 270: W08 Lab: Regression Testing

Implement your regression test plan and report on the results.

Implement the Plan

If you received any feedback on your regression test plan, please make the necessary changes to ensure it is of sufficient quality. Proceeding with a flawed plan is likely to result in flawed results.

With your partner(s), implement your regression test plan. This includes the following activities:

  1. Pre-Test: Conduct all the test cases and report on the results. Hopefully, each test will pass.
  2. Code Change: Make the planned change to the code. Note that this must be done after the pre-tests are run.
  3. Post-Test: Conduct all the test cases again. Hopefully, all the tests will pass this time too.

Report

Generate a report recording all that transpired. This report must include details of the code change and the result of the test results.

Code Changes

Present the code before the change and after the change. In both cases, only present the code that was changed; do not just insert the entire code-base.

For example, if I were to change a tic-tac-toe board from being a 3x3 array of characters to a 3x3 array of integers, I would include the following:

  • The before and after of the board declaration.
  • Any code necessary to convert from a character (in the file format) to an integer (for the new data structure).
  • Every instance in the code that referenced the board (both before and after the change was made).

Note: if your code change has far-reaching implications (such as the tic-tac-toe board which may be referenced a dozen times), then the number of test cases may be large. In other words, choose your code change carefully!

Test Cases

The report will consist of one section per test case. Each section will consist of the following:

  • The test case details. In most cases, this will be straight from the plan submitted earlier in the week. However, if a change in the test case is made, then the updated test case will be presented here.
  • The pre-test results.
  • The post-test results.

Submission

Your team will submit a single document for your test cases. The document needs to be in the PDF format.
