CSE 270: W08 Overview

Regression testing is a special type of testing performed after a code change has been made to the product. The goal is to verify that the code change does not impact the existing functionality of the system. In other words, the goal is to make sure that nothing was broken.

Objectives

By the end of the week, each student will be able to:

  • Define regression testing and explain how it is a necessary part of the software development process.
  • Appreciate the importance of regression testing.
  • Perform a regression test.

Plan of the Week

This week, we will complete the following activities:

  1. Reading: Read about regression testing.
  2. Identify a Codebase: With your partner(s), identify a codebase suitable for this week's lab.
  3. Create a Regression Test Plan: Create a plan for performing regression testing on your codebase of choice.
  4. Conduct a Regression Test: Carry out your regression test plan and report on the results.
