CSE 270: W12 Overview

Testing is a fundamental tool used to prevent project failure. One challenge is that project failure can come from many sources. In order to provide the greatest possible protection against project failure, it is necessary to look at all the potential failure sources. Up to this point, we have been focusing on technical failure and usability failure. This week, we will learn about other issues which may arise when working on software products.

Objectives

By the end of the week, each student will be able to:

  • Identify security, legal, and other sources of risk.
  • Appreciate different components of software project risk.

Plan of the Week

This week, we will complete the following activities:

  1. Reading: Read two chapters.
  2. Risk Identification: With your partner(s), create list of the possible risks to a project.
  3. Risk Mitigation: With your partner(s), come up with ways to manage the previously identified risks.
