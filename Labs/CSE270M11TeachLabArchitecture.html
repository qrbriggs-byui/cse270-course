CSE 270 Teach Lab M11: Architecture

Lab 11 - Architecture Testing

Overview

One of the most important aspects of any application is the architecture, knowing where the information is being collected, processed, transferred, and stored, typically to identify stages and states found in architecture.
Your team needs to create tests to identify where to pass data along the different communication paths and determine the use of communication services.

Assignment

For this lab, you will determine what data to pass along the different communication paths.

Communication Paths

Pick one of the following communication paths, and determine what data to pass and what resources to use.

  • Communication between External Device (Monitor) and Dr. Mobile. (Peer-To-Peer).
  • Communication between one Dr. Mobile Device and another Device(Peer-To-Peer).
  • Communication between Transportation and Dr. Mobile (Wireless).
  • Communication between Home Base and Dr. Mobile (Client - Server).
  • Communication between Supply Chain and Dr. Mobilee (Rule-Based).
  • Communication between User and Dr. Mobile HMI(Event-Driven).
  • Communication between Dr. Mobile to Database (Data).

Workflow

Using the workflow (Lab 08) of Dr. Mobile, identify the needed data for each step, created during that step, and the next step's output.

Log Files

Each element of the workflow and architecture elements need to record the transition states and data. Answer the following questions to help identify logging information:

  • What is needed to be able to track the workflow through the entire system?
  • What is a normal output to the log file? To the user?
  • What is an error output to the log file? To the user?
  • What is a warning output to the log file? To the user?
